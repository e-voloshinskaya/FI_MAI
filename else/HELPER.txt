Как создать удаленный репозиторий + локальную версию:
1. Удаленный: просто нажать "создать репозиторий", ввести имя и добавить ридми.
2. Локальный:
	а. В папке-родителе для создаваемого локального репозитория ввести команду
		git clone [полный адрес https://github.com/username/reponame.git]
	б. Сконфигурировать имя пользователя и почту, см. "Почему contributions не считаются?"
	в. Сделать первый коммити пуш. Ура. 


Как собирать программу из нескольких файлов?
1. gcc -c f1.c f2.c ... fn.c. !!! Заголовочные файлы, .h, не включаются в список !!!
2. Будут созданы f1.o ... fn.o
3. gcc -o [название программы] f1.o ... fn.o
4. Получим исполняемый файл [название программы]
5. Запускаем как обычно ./test [параметры]


Как отправить изменения в гитхаб?
1. git status - узнаем, в каком состоянии локальный репозиторий относительно основного, онлайн
2. git pull, если локальный репозиторий содержит устаревшие данные
3. git add [путь к файлу из тек папки]
4. git commit -m "Комментарий 1"
5. git --amend (-m "Комментарий 2"), чтобы внести изменения в коммит
6. git push и ввести логин-пароль

git add параметры:
. - добавить все, кроме игнорируемых (т.е. changed, untracked и removed)
-u - все, кроме untracked и ignored
-f . - все, включая игнорируемые


Почему contributions не считаются?
1.Если не установлен e-mail адрес для коммитов (глобально или локально)
Если локально,
ввести в ком. строке
$ git config user.email "email@example.com"
проверить:
$ git config user.email
email@example.com [появится на экране, если e-mail установлен]
